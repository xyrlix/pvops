## 一、项目定位

### 1.1 一句话定位

**“面向新能源资产运营商的 AI 运营决策平台”**：先帮电站看清风险、挽回损失，再帮储能套利增收，最终参与电力市场交易，让资产越用越值钱。

### 1.2 核心差异化

| 维度 | 智子熹源 V3.2 | 本方案 |
| --- | --- | --- |
| 业务范围 | 仅光伏运维 | 光伏运维 → 储能策略 → 电力交易 |
| AI 角色 | 诊断助手 | 诊断 + 策略生成 + 交易决策 |
| 客户端 | PC 大屏为主 | PC + H5 + 小程序 + APP |
| 部署形态 | 公有云/私有化/混合 | 主打私有化，数据不出域 |
| 价值量化 | 口号式指标 | 每笔 Action 可核算收益 |

## 二、最终技术栈

| 层级 | 技术选型 | 选型理由 |
| --- | --- | --- |
| **PC Web / H5** | Vue 3 + Vite + TypeScript + Pinia + Vue Router | 与小程序技术栈统一，维护简单，国内生态成熟 |
| **UI 组件库** | Element Plus 或 Arco Design | 中后台系统组件丰富 |
| **图表** | ECharts（vue-echarts） | 大屏原生支持 |
| **表格** | Vxe-Table 或 AG Grid Vue | 高性能表格 |
| **小程序** | Uni-app（Vue 3） | 一套代码覆盖微信/支付宝/抖音 |
| **APP** | Uni-app 打包 / Flutter（后期） | 前期轻量，后期按需升级 |
| **后端** | FastAPI（Python 3.11+） | AI/IoT/算法生态最强，异步高性能 |
| **任务队列** | Celery + Redis | 简单可靠 |
| **时序数据库** | TDengine 3.x | 国产、光伏场景多、压缩比高 |
| **业务数据库** | PostgreSQL 16 | 稳定，支持 pgvector |
| **缓存/会话** | Redis 7 | 高性能 |
| **向量库** | PGVector（起步） | 低门槛，避免早期引入 Milvus |
| **MQTT Broker** | Mosquitto | 轻量够用 |
| **对象存储** | MinIO（可选）/ 本地目录 | 小客户用本地目录即可 |
| **AI 框架** | LangChain + RAG | 成熟，小团队可控 |
| **大模型** | DeepSeek / 通义千问（API），后期本地化 | 国内便宜，私有化路径清晰 |
| **优化算法** | PuLP / CVXPY / OR-Tools | 储能/交易优化够用 |
| **部署** | Docker Compose | 维护简单 |
| **远程访问** | Cloudflare Tunnel | 无需公网 IP，自动 HTTPS |
| **静态文件服务** | FastAPI StaticFiles | 省掉 Nginx/Caddy，极简部署 |
| **版本控制** | Git + GitHub/GitLab | 标准方案 |
| **CI/CD** | GitHub Actions / GitLab CI | 自动化测试与部署 |

### 3.2 容器清单

| 容器 | 用途 | 是否必需 |
| --- | --- | --- |
| `backend` | FastAPI（API + 静态前端 + AI 模块） | ✅ |
| `worker` | Celery Worker（可与 backend 合并） | ✅ |
| `scheduler` | Celery Beat 定时任务（可与 worker 合并） | ✅ |
| `tdengine` | 时序数据库 | ✅ |
| `postgres` | 业务数据库 | ✅ |
| `redis` | 缓存 + 任务队列 + 会话 | ✅ |
| `mosquitto` | MQTT Broker | ✅ |
| `cloudflared` | 远程访问隧道 | ✅（无公网 IP 时） |
| `minio` | 对象存储 | ⚠️ 可选 |

**最简可压到 6 个容器**：backend（含 worker/scheduler）+ tdengine + postgres + redis + mosquitto + cloudflared。

## 四、安全基线

### 4.1 认证与授权

- JWT + Refresh Token 认证
- 密码 Argon2id 哈希
- RBAC 权限模型：超级管理员 / 集团管理员 / 场站管理员 / 运维人员 / 只读用户
- 数据行级隔离：用户只能看到自己权限范围内的场站数据
- 支持多租户（后期）

### 4.2 网络安全

- Cloudflare Tunnel 提供 TLS 加密，无需暴露内网端口
- 内部容器间通信使用 Docker 内部网络
- 生产环境禁用 `root` 运行容器
- API 全局限流（按 IP + 用户）

### 4.3 数据安全

- 传输加密：Cloudflare Tunnel TLS
- 敏感配置（数据库密码、API Key）使用 Docker Secrets 或环境变量注入，禁止硬编码
- 数据库定期自动备份（PostgreSQL + TDengine）
- 备份文件加密存储

### 4.4 控制安全

- 储能充放电策略、电力交易申报等控制类指令**默认建议模式**
- 自动执行需客户书面授权 + 白名单 + 双因子确认
- 所有控制操作记录审计日志，保留 ≥ 180 天
- 支持操作回滚

### 4.5 安全扫描

- Python：`bandit` + `safety`
- Node：`npm audit`
- 依赖漏洞纳入 CI 门禁

## 五、测试与质量保障

### 5.1 测试分层

| 层级 | 工具 | 覆盖范围 |
| --- | --- | --- |
| 单元测试 | pytest（后端）、Vitest（前端） | 核心算法、工具函数、组件 |
| 集成测试 | TestContainers + FastAPI TestClient | API、数据库、时序库交互 |
| E2E 测试 | Playwright | 关键用户流程 |
| 安全测试 | bandit / safety / npm audit | 依赖和代码漏洞 |
| 性能测试 | locust / k6 | API 并发、数据库写入 |

### 5.2 设备数据模拟器

MVP 期必须建立设备模拟器，支持：
- 生成虚拟电站的分钟级运行数据
- 模拟晴天/阴天/雨天/遮挡/故障等场景
- 批量生成 1 个月以上历史数据
- 实时推送模拟数据到 MQTT

### 5.3 CI/CD 门禁

```yaml
# 后端
- ruff lint
- mypy 类型检查
- pytest 单元测试（覆盖率 ≥ 60%，核心算法 ≥ 80%）
- bandit 安全扫描
- safety 依赖检查

# 前端
- ESLint
- Prettier 格式检查
- vue-tsc 类型检查
- Vitest 单元测试
- npm run build
```

## 六、开发流程

### 6.1 迭代节奏

- **2 周一个 Sprint**
- **每周一个内部版本**
- **每 2 周一个客户可见版本**
- **每日 15 分钟站会**（可异步）

### 6.2 Git 工作流

- `main`：生产分支
- `develop`：开发分支
- `feature/*`：功能分支，生命周期 < 3 天
- `hotfix/*`：紧急修复
- 所有合并通过 PR，至少 1 人 Review，CI 通过

### 6.3 提交规范

```
feat: 新功能
fix: 修复
docs: 文档
test: 测试
refactor: 重构
chore: 杂项
```

### 6.4 环境

| 环境 | 用途 | 数据 |
| --- | --- | --- |
| local | 开发调试 | 模拟数据 |
| test | 自动化测试 | 模拟数据 |
| staging | 客户预览 | 真实数据（脱敏） |
| prod | 生产 | 真实数据 |

## 八、数据需求

### 8.1 必需数据

| 数据类型 | 字段 | 频率 | 来源 |
| --- | --- | --- | --- |
| 逆变器数据 | 功率、电压、电流、发电量、状态、故障码 | 1~15 分钟 | 厂家云平台 / 本地 SCADA |
| 气象数据 | 辐照量、环境温度、风速 | 1~15 分钟 | 气象站 / 厂家平台 |
| 电表数据 | 上网电量、下网电量、实时功率 | 1~15 分钟 | 电表 / SCADA |
| 电站信息 | 容量、并网时间、组件型号、经纬度 | 一次性 | 客户提供 |
| 设备台账 | 逆变器型号、序列号、组串配置 | 一次性 | 客户提供 |

### 8.2 可选数据

| 数据类型 | 场景 |
| --- | --- |
| 组串级电流/电压 | 组串离散率、精准定位故障 |
| 储能 BMS/PCS | 阶段二储能策略 |
| 负荷数据 | 需量管理、套利策略 |
| 历史故障记录 | AI 训练、知识库 |

### 8.3 数据文件格式

- CSV 首选
- 时间戳统一：`YYYY-MM-DD HH:MM:SS`
- 字段名英文，避免中文列名
- 每个文件附带《字段说明文档》

## 十三、关键决策记录

| 决策 | 结论 | 原因 |
| --- | --- | --- |
| 前端框架 | Vue 3 + Vite | 与 Uni-app 统一，维护简单 |
| 后端框架 | FastAPI (Python) | AI/IoT/算法生态最强 |
| 部署方式 | Docker Compose + Cloudflare Tunnel | 极简，无需公网 IP |
| 反向代理 | 不需要 Caddy/Nginx | Cloudflare Tunnel + FastAPI StaticFiles 足够 |
| 数据库 | TDengine + PostgreSQL + Redis | 时序/业务/缓存分离 |
| MQTT | Mosquitto | 轻量够用 |
| 消息队列 | Redis + Celery | 无需 Kafka |
| 微服务 | 不需要 | MVP 期单体足够 |
| SSR | 不需要 | 内网系统无需 SEO |
| AI 路径 | 规则兜底 + RAG + LLM 解释 | 准确率可控，逐步迭代 |
| 控制指令 | 默认建议模式 | 安全第一 |

## 十四、结语

本方案以**“维护简单、可控、稳定够用、不过度设计”**为核心原则，确定：

- **前端**：Vue 3 + Vite + Uni-app
- **后端**：Python FastAPI
- **部署**：Docker Compose + Cloudflare Tunnel
- **阶段**：先光伏运维 MVP（12 周），再储能策略（10 周），最后电力交易（12 周+）
- **质量**：规则 + AI 双保险，模拟器 + CI 保障测试

**本文件作为项目实施的唯一基准，后续开发、交付、迭代均以此为准。**
