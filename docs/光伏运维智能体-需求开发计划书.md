## 一、原方案解析（智子熹源 V3.2 全量拆解）

### 1.1 对方卖的是什么

* **厂商**：智子熹源（上海）科技有限公司，定位"能碳数智化、AI 大模型技术服务商"

* **核心产品**："光伏运维专家智能体"，本质是 **知识库 + 大模型 + 智能体协同**

* **战绩**：已接入 1200+ 分布式场站、2GW+ 装机

* **三大价值指标**：管理效率 ↑25 倍+、等效发电小时数 ↑5%~10%、综合收益 ↑5%~8%

### 1.2 对方的核心叙事（5 段式）

1. **行业背景**：双碳驱动装机激增 → 传统被动式运维失效（看不清 / 算不准 / 管不好）

2. **行业特点**：小 · 散 · 异 · 弱 · 高 → 数据难治理 / 异常难识别 / 损失难量化 / 经验难复用 / 决策难闭环

3. **工具演进三阶段**：SCADA(被动) → 单点诊断(分散) → **专家智能体(自主闭环)** ← 对方核心卖点

4. **三层视角落地**：管理层(场站级 / 日) → 分析层(逆变器级 / 月) → 执行层(组串级)

5. **学习反馈机制**：反馈记忆增强(当前) → 小模型离线训练(演进) → 大模型在线学习(演进)

### 1.3 对方的功能矩阵（我们要复刻的全部能力）

| 视角  | 粒度  | 时间维度 | 核心能力                                                          | UI 形态                 |
| --- | --- | ---- | ------------------------------------------------------------- | --------------------- |
| 管理层 | 场站  | 日    | 异常汇总 / 高风险标注 / 运维优先级、AI 洞察自动汇总、单站诊断报告生成                       | 场站气泡图 + 场站列表 + 优先级排序  |
| 管理层 | 场站  | 月    | 月度健康度走势、周期性异常识别、健康 / 亚健康 / 异常三色分区                             | 月度健康度热力图              |
| 分析层 | 逆变器 | 月    | 群组基线对比、低效组串诊断、设备级 AI 深度分析（根因 + 证据链 + 建议）                      | 群组对比图 + 右侧 AI 对话框     |
| 执行层 | 组串  | 实时   | 组串异常 AI 诊断、智能分析、专家建议、可落地修复路径、全流程诊断（数据校验 → 异常识别 → 根因拆解 → 修复建议） | 组串级诊断卡 + AI 对话 + 工单生成 |
| 横切  | 全部  | -    | 知识问答、故障报告自动生成、Action Guide、优先级待办、工单联动、反馈记忆                    | 对话框 + 工单流             |

### 1.4 关键产品差异化点（我们能做到"比他更好"的地方）

| 维度    | 智子熹源 V3.2       | 我们（计划）                                                              |
| ----- | --------------- | ------------------------------------------------------------------- |
| 范围    | 仅光伏运维           | 光伏运维为切入，**架构预留储能 EMS + 电力交易**，向 VPP 演进                              |
| 诊断粒度  | 场站 → 逆变器 → 组串   | 同等粒度，**额外加组件级**（如有组件级数据）                                            |

| 储能    | 文档未体现           | **二期落地充放电策略优化**（峰谷套利、需量管理）                                          |
| 电力交易  | 文档未体现           | **三期接入虚拟电厂 / 现货市场**（报量报价、收益优化）                                      |
| 部署    | 公有云 / 私有化 / 混合云 | 主打**私有化**（数据不出域，业主更愿意买单）                                            |

## 四、详细功能需求清单（按里程碑组织）

### 🎯 里程碑 1（M1-M2，第 1-2 月）：数据采集 + 分析 + MVP 大屏

#### 模块 1：数据采集网关（L1）

**目标**：把电站"接进来"，先把数据洗干净地存下来。

* **F1.1 协议适配器**

  * 逆变器：华为 SUN2000、阳光 PowerStation、固德威、锦浪、上能、古瑞瓦特（Modbus TCP/RTU + 厂家云 API 双通道）

  * 气象站：辐照 / 温度 / 风速 / 组件背板温度（Modbus/MQTT）

  * 电表：关口表 / 计费表（DL/T645、Modbus、IEC104）

  * 储能：BMS/PCS（Modbus TCP / 厂家 API）— *二期启用，本期接入预留*

* **F1.2 边缘采集程序**：Python + asyncio，部署在站端网关（树莓派 / 工控机），断点续传、本地缓存、压缩上报

* **F1.3 数据治理**：时间戳对齐、缺失插值、异常值剔除、单位归一、设备映射字典

* **F1.4 实时消息管道**：EMQX(MQTT) → Kafka → 流处理

* **交付物**：可接入至少 3 个品牌逆变器 + 气象站 + 电表的采集程序，1 秒~1 分钟可配采集周期

#### 模块 2：数据存储与建模（L2）

* **F2.1 时序库建模**（TDengine）：超级表 - 逆变器 / 组串 / 气象 / 电表 / 储能，标签 = 场站 + 设备 ID

* **F2.2 关系库建模**（PostgreSQL）：场站、设备、组串、用户、权限、告警、工单、报告

* **F2.3 向量库**（PGVector 起步，后迁 Milvus）：知识库文档向量化、历史诊断案例

* **F2.4 对象存储**（MinIO）：图片（红外 / 巡检照片）、报告 PDF、配置文件

#### 模块 3：指标计算引擎（L3）

对标文档"一眼看清、问题指哪打哪"，需要这些核心指标：

| 指标类 | 具体指标                                    |
| --- | --------------------------------------- |
| 性能  | 等效发电小时数、PR(性能比)、系统效率、设备效率               |
| 健康  | 场站健康度、逆变器健康度、组串离散率、月度健康走势               |
| 损失  | 总损失、限电损失、故障损失、辐照损失、效率损失（**量化到 kWh 和元**） |
| 异常  | 低效组串、电流 / 电压异常、温度异常、通讯中断                |
| 经济  | 日 / 月 / 年发电量、上网电量、理论收益、实际收益、损失折算金额      |

* **F3.1 实时计算**（流式）：每分钟聚合

* **F3.2 离线计算**（批处理）：日 / 月 / 年 rollup

* **F3.3 基线对比**：群体基线（同区域 / 同厂家 / 同容量对比），个体基线（自身历史）

#### 模块 4：异常识别与健康度（L3）

* **F4.1 规则引擎**：阈值 / 趋势 / 离散率 / 通讯中断 / 功率突降等 20+ 内置规则

* **F4.2 健康度模型**：0-100 分，按"健康 / 亚健康 / 异常"三色分区（对标文档配色）

* **F4.3 异常去重聚合**：避免告警风暴，同根因聚合

#### 模块 5：前端大屏与工作台（L6）⭐ MVP 核心展示面

**对标文档 P7-P12 的 UI，我们 1:1 复刻并优化：**

* **F5.1 集团总览大屏**（对标 P7 气泡图）

  * 横轴 = 装机容量，纵轴 = 发电完成率，气泡大小 = 损失金额，颜色 = 健康度

  * 右侧高风险场站 TOP 榜，点击下钻

  * 顶部 AI 洞察：今日异常自动汇总 + 优先级建议

* **F5.2 场站看板**（对标 P7-P8）

  * 场站实时功率曲线、辐照-发电对照、损失瀑布图

  * 月度健康度热力图（日 × 场站，三色，对标 P8）

* **F5.3 设备分析工作台**（对标 P9-P10）

  * 群组对比：选同类逆变器横向排名

  * 逆变器详情：功率曲线、组串离散率分布

  * 右侧 AI 对话框：点击"诊断"触发 AI 分析

* **F5.4 组串诊断卡**（对标 P11-P12）

  * 组串性能对比图、I-V 曲线（如可获取）

  * AI 诊断卡：根因 + 证据链 + 建议 + 可执行 Action

* **F5.5 工单中心**：Action 自动转工单，状态流转，闭环跟踪

#### 模块 6：AI 诊断智能体（L5）⭐ 差异化核心

* **F6.1 诊断 Agent**：输入异常 ID/设备 ID，自动跑"数据可用性校验 → 异常识别 → 根因拆解 → 修复建议"全流程

* **F6.2 报告 Agent**：日 / 周 / 月报、单站诊断报告自动生成（PDF）

* **F6.3 问答 Agent**：基于光伏运维知识库的 RAG 问答（"XX 故障怎么处理？"）

* **F6.4 工单 Agent**：根据诊断结论自动生成工单，分配优先级

* **F6.5 证据链输出**：每个结论都附"为什么"——数据截图、对比基线、规则命中

* **F6.6 反馈记忆**：人工纠错 → 案例沉淀 → Prompt 增强（对标 P13）

#### 模块 7：知识库（L5）

* **F7.1 预置知识**：光伏运维手册、故障代码库、设备说明书、行业规范

* **F7.2 自定义知识**：业主私有运维 SOP、历史案例

* **F7.3 RAG 检索**：向量 + 关键词混合检索

**M1-M2 交付物**：可接入 3 品牌设备的真实电站，跑通"采集 → 指标 → 大屏 → AI 诊断 → 工单"全链路，能在客户现场演示。

### 🎯 里程碑 2（M3-M4，第 3-4 月）：储能充放电策略 ⭐ 差异化壁垒

> 这是对方文档**完全没有的**能力，也是我们拉开差距的关键。

#### 模块 8：储能接入与监控

* **F8.1 BMS/PCS 数据接入**：SOC、SOH、电压、温度、充放电功率

* **F8.2 储能看板**：实时状态、充放电曲线、循环次数、可用容量

#### 模块 9：预测模型

* **F9.1 发电预测**：基于气象 + NWP，短期(0-24h) + 超短期(15min-4h)

* **F9.2 负荷预测**：工商业业主用电负荷预测

* **F9.3 电价预测**：分时电价、现货价格（三期深度用）

#### 模块 10：充放电策略优化（核心算法）

* **F10.1 峰谷套利**：低充高放，最大化套利收益

* **F10.2 需量管理**：削峰，降低基本电费

* **F10.3 备用容量**：保留安全裕度

* **F10.4 多目标优化**：套利 + 需量 + 寿命损耗的混合整数规划（PuLP/CVXPY 求解器）

* **F10.5 策略下发**：通过 EMS 协议下发到 PCS，或输出建议给运维人员确认

* **F10.6 效果复盘**：每日对比"策略收益 vs 实际执行"，持续优化

**M3-M4 交付物**：含储能的电站可自动跑充放电策略，输出每日收益提升报告。

***

### 🎯 里程碑 3（M5-M6，第 5-6 月）：电力交易 / 虚拟电厂 ⭐ 顶级能力

#### 模块 11：电力市场接入

* **F11.1 现货市场数据接入**：节点电价、负荷预测、新能源出力预测

* **F11.2 报量报价**：基于发电预测 + 储能状态，生成申报曲线

* **F11.3 结算对账**：交易结果回溯、收益分摊

#### 模块 12：虚拟电厂聚合

* **F12.1 多场站聚合**：把分散的新能源 + 储能 + 可调负荷聚合成"可调度资源"

* **F12.2 调度响应**：响应电网调度指令（AGC、调频、需求响应）

* **F12.3 收益分配**：按贡献度分配聚合收益

#### 模块 13：AI 交易智能体

* **F13.1 交易 Agent**：基于市场 + 预测 + 策略，自动生成申报方案（人审核后执行）

* **F13.2 风险控制**：亏损预警、回撤控制

**M5-M6 交付物**：具备参与现货市场 + 需求响应的能力，配套交易看板和 AI 辅助决策。

## 五、技术选型（已锁定）

| 层     | 技术                                                             | 选型理由                     |
| ----- | -------------------------------------------------------------- | ------------------------ |
| 前端    | **Next.js 14 + TypeScript + Tailwind + shadcn/ui**             | 全栈一体化，SSR 性能好，组件生态丰富     |
| 图表    | **ECharts**（大屏）+ **AG Grid**（表格）+ **Apache ECharts GL**（3D 气泡） | 大屏视觉效果对标对方，气泡图 / 热力图原生支持 |
| 后端    | **Python 3.11 + FastAPI + Uvicorn**                            | AI 生态无缝，异步高性能，自动文档       |
| 任务队列  | **Celery + Redis**                                             | 异步任务、定时调度                |
| 时序库   | **TDengine 3.x**                                               | 国产、光伏场景大量案例、压缩比高         |
| 关系库   | **PostgreSQL 16**                                              | 成熟稳定，支持 pgvector         |
| 向量库   | **PGVector（起步）→ Milvus（规模化）**                                  | 起步低门槛，后期可平滑迁移            |
| 缓存    | **Redis 7**                                                    | 实时指标缓存、会话                |
| 消息队列  | **EMQX（MQTT）+ Kafka**                                          | 边缘接入 + 流处理               |
| 对象存储  | **MinIO**                                                      | 私有化部署友好，S3 兼容            |
| AI 框架 | **LangChain + LangGraph**                                      | 多 Agent 编排，业界标准          |
| 大模型   | **DeepSeek V3 + 通义千问 + Claude/GPT（备选）**                        | 国内私有化首选 DeepSeek，API 起步  |
| 算法    | **NumPy/Pandas/Scikit-learn + PuLP/CVXPY**                     | 预测 + 优化求解                |
| 部署    | **Docker Compose（小客户）→ K8s（大客户）**                              | 私有化交付标准                  |

## 九、差异化竞争策略（怎么"比他做得更好"）

1. **范围更广**：他只做光伏运维，我们**光伏 + 储能 + 交易一体化**，单点切入、平台演进

2. **决策更闭环**：他的 AI 止步于"诊断建议"，我们**直通工单 + 策略下发 + 交易执行**

3. **效果更可量化**：每个 AI 建议都附**收益预估**（"按建议处理可挽回 X 度电 = Y 元"）

4. **知识资产化**：他的反馈是黑盒，我们做**可视化案例库**，业主可看到自己的运维知识沉淀

5. **开放生态**：他封闭，我们**开放 API**，方便业主对接既有 ERP/MES

6. **价格更灵活**：他主打订阅，我们**私有化买断 + 年服务费 + 效果分润**三选一
