## 三、推荐 Roadmap（阶段一 MVP，3 个月）

> 按“先数据、再指标、再大屏、最后 AI 增强”的顺序，每 2 周交付一个可演示价值点。

### Sprint 1：数据底座（W1-W2）

- [x] 引入 TDengine 仓库抽象与 SQLite fallback，设计逆变器/气象/电表超级表
- [x] 设计统一 Device 模型：逆变器、气象站、电表、组串，建立 Station→Device→String 拓扑
- [x] 采集器协议适配层骨架（Modbus TCP/RTU + Simulator，厂家云 API 接口可扩展）
- [x] 边缘采集器 Python 骨架（asyncio + pymodbus），支持本地缓存与断点续传
- [x] Docker Compose 开发环境（PostgreSQL + TDengine + Redis + Mosquitto）

**验收**：
- [x] 模拟器数据可稳定写入 SQLite fallback
- [x] 切换 `TSDB_BACKEND=tdengine` 即可将数据写入 TDengine（需 Docker 环境）
- [x] 前端可展示设备拓扑与设备清单

### Sprint 2：核心指标引擎（W3-W4）

- [ ] 等效发电小时数、系统效率、设备效率
- [ ] 损失分解：总损失、限电损失、故障损失、辐照损失、效率损失
- [ ] 收益/损失折算：理论收益、实际收益、损失金额（元）
- [ ] 群体基线对比（同容量电站横向排名）
- [ ] 逆变器群组排名、组串离散率计算

**验收**：指标计算误差 < 3%，可输出日/月汇总。

### Sprint 3：高级可视化大屏（W5-W6）

- [ ] 集团总览气泡图（ECharts GL 或 2D 散点）
- [ ] 高风险场站 TOP 榜 + 今日 AI 洞察汇总
- [ ] 月度健康度热力图
- [ ] 场站看板：辐照-发电对照图、损失瀑布图
- [ ] 设备分析页：逆变器群组对比、组串离散率分布

**验收**：UI 对标 P7-P12，关键大屏可演示。

### Sprint 4：AI Copilot + 知识库（W7-W8）

- [ ] 右侧 AI 对话组件，支持点设备触发诊断
- [ ] 知识库上传（PDF/Word/SOP）+ PGVector 向量化
- [ ] RAG 问答 Agent
- [ ] 诊断报告 PDF 导出
- [ ] 诊断反馈（正确/部分/错误）记录

**验收**：可针对告警/设备进行对话式诊断，可导出 PDF 报告。

### Sprint 5：工单与权限闭环（W9-W10）

- [ ] 角色权限真正生效（RBAC 路由/按钮级）
- [ ] 多租户数据隔离（Tenant/Station 行级过滤）
- [ ] 告警去重聚合、优先级排序
- [ ] 工单派单、执行反馈、案例沉淀
- [ ] 私有化部署脚本与 README

**验收**：不同角色看到不同数据，可私有化一键部署。

### Sprint 6：MVP 打磨与验收（W11-W12）

- [ ] 端到端真实电站接入演示
- [ ] 性能优化（查询、图表渲染）
- [ ] 操作手册、部署文档
- [ ] 客户试用与反馈收集

## 五、下一步需要你拍板

1. **是否按上述 6 个 Sprint 推进？** 还是把某些能力（如储能）提前？
2. **是否有真实电站可作为试点？** 设备品牌和网络条件？
3. **是否引入 TDengine 容器替换 SQLite？** 这是数据底座的关键决策。
4. **AI 大模型当前继续用规则 + API，还是尽快接入本地 DeepSeek/Qwen？**

建议 **立即开始 Sprint 1**，先把数据底座和真实设备接入跑通，否则大屏和 AI 都是“无米之炊”。
