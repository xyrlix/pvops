# 电站数据采集需求清单

> **用途**：为光伏运维智能体 MVP 提供历史训练数据与实时接入参考  
> **目标读者**：电站业主、运维人员、设备厂家对接人  
> **最低要求**：至少 1 个月连续历史数据，分钟级最佳，小时级可用  
> **编制日期**：2026-06-23

---

## 一、我们需要哪些数据？

### 1.1 逆变器运行数据（必需，优先级最高）

逆变器是光伏电站最核心的数据源，直接决定发电量计算、异常识别、AI 诊断的准确性。

| 字段名 | 说明 | 用途 | 是否必需 |
| --- | --- | --- | --- |
| 时间戳 | 每条记录的采集时间 | 时序对齐、趋势分析 | ✅ 必需 |
| 逆变器编号 | 厂家设备编码或自定义名称 | 设备识别 | ✅ 必需 |
| 有功功率 | 实时输出功率（kW） | 发电量、PR、异常识别 | ✅ 必需 |
| 日发电量 | 当日累计发电量（kWh） | 收益核算、完成率 | ✅ 必需 |
| 总发电量 | 逆变器投运以来累计发电量（kWh） | 生命周期分析 | ✅ 必需 |
| 直流电压 | PV 侧电压（V） | 组串状态、故障判断 | ✅ 必需 |
| 直流电流 | PV 侧电流（A） | 组串状态、离散率 | ✅ 必需 |
| 交流电压 | 并网侧电压（V） | 电网异常判断 | ⚠️ 建议有 |
| 交流电流 | 并网侧电流（A） | 负载/并网状态 | ⚠️ 建议有 |
| 交流频率 | 电网频率（Hz） | 电网质量分析 | ⚠️ 建议有 |
| 逆变器状态 | 运行/待机/故障/告警 | 状态监控、告警触发 | ✅ 必需 |
| 故障码 | 厂家定义的故障代码 | AI 根因分析 | ⚠️ 建议有 |
| 温度 | 逆变器内部/散热器温度 | 热管理、效率分析 | ⚠️ 建议有 |

**采样频率要求**：
- 🥇 首选：1~5 分钟级
- 🥈 可用：15 分钟级
- 🥉 最低：小时级（仅能做基础指标）

## 二、数据从哪里获取？

### 方式一：从逆变器厂家云平台导出（推荐，最快）

绝大多数逆变器品牌都有云平台，数据已经采集好，可直接导出。

| 逆变器品牌 | 云平台名称 | 导出方式 | 备注 |
| --- | --- | --- | --- |
| 华为 | FusionSolar | Web 后台导出 Excel/CSV，或开放 API | 数据较规范 |
| 阳光电源 | iSolarCloud | Web 后台导出，或开放 API | 数据较规范 |
| 固德威 | SEMS Portal | Web 后台导出 | 需登录业主账号 |
| 锦浪 | SOLARMAN / 锦浪云 | Web 后台导出 | 需业主授权 |
| 古瑞瓦特 | ShineServer / 古瑞瓦特云 | Web 后台导出 | 需业主授权 |
| 上能电气 | 上能云 | Web 后台导出 | 需业主授权 |
| 科士达 | 科士达云 | Web 后台导出 | 需业主授权 |

**操作步骤**：
1. 登录对应厂家云平台
2. 进入“数据查询”或“报表中心”
3. 选择时间范围（建议最近 1~3 个月）
4. 选择设备（逆变器/组串/气象站/电表）
5. 导出 CSV 或 Excel

---

### 方式二：从电站本地 SCADA/EMS 系统导出

如果电站有本地监控系统，可直接从本地数据库导出。

**常见数据源**：
- 本地 SCADA 系统
- 第三方物联网平台（如 ThingsBoard、组态王）
- 电站自有数据库（MySQL/InfluxDB/TDengine 等）

**需要电站方配合**：
- 提供数据库只读账号，或
- 由电站运维人员按模板导出 CSV

---

### 方式三：临时部署采集网关（适合长期实时采集）

如果需要更高实时性的数据，可在电站本地部署一台采集网关（工控机/树莓派）。

**要求**：
- 能接入电站内网或设备通信网络
- 支持 Modbus TCP/RTU、MQTT、IEC104 等协议
- 有稳定的网络出口上报数据

**此方式适合作为正式运行方案，但不适合快速获取历史数据。**

## 三、数据文件格式要求

### 3.1 推荐文件格式

- **CSV 文件**：最通用，便于程序读取
- **Excel 文件（.xlsx）**：可人工查看，但程序读取稍麻烦
- **JSON/API 接口**：适合厂家云 API 直接对接

### 3.2 CSV 文件标准格式

```csv
timestamp,inverter_id,active_power_kW,daily_energy_kWh,total_energy_kWh,dc_voltage_v,dc_current_a,inverter_status,fault_code
2026-05-01 00:00:00,INV-001,0,1250.5,152300.2,0,0,Standby,0
2026-05-01 00:05:00,INV-001,12.5,1250.8,152300.5,620,18.2,Running,0
2026-05-01 00:10:00,INV-001,15.2,1251.3,152301.0,625,19.5,Running,0
```

### 3.3 字段命名建议

- 时间戳统一用 `timestamp`
- 设备 ID 统一用 `device_id` 或 `inverter_id`
- 功率统一用 `active_power_kW`
- 发电量统一用 `daily_energy_kWh`、`total_energy_kWh`
- 避免中文列名、避免空格、避免特殊符号

## 四、数据质量要求

### 4.1 时间连续性

- 数据应尽量连续，**缺失率不超过 5%**
- 如果缺失严重，需说明缺失原因和时间段

### 4.2 时间覆盖

- **最低要求**：连续 1 个月
- **建议**：连续 3 个月（覆盖晴天、阴天、雨天等多种天气）
- **最佳**：1 年以上（可分析季节性规律）

### 4.3 数据对应关系

- 逆变器数据、气象数据、电表数据的时间戳应对齐
- 同一电站的设备编码应统一，避免同一设备多个名字

## 六、常见问题

### Q1：拿不到分钟级数据，只有小时级，能用吗？

**能用，但效果受限。** 小时级数据可以计算日发电量、日 PR、粗略异常，但无法识别短时遮挡、通讯中断等分钟级问题。建议同时尽量获取分钟级数据。

### Q2：气象站数据没有怎么办？

- 优先从厂家云平台或本地 SCADA 导出
- 如果确实没有，可用公开气象数据源（如 NASA POWER、Solcast、中国气象数据网）作为替代，但精度会下降
- 长期建议加装气象站

### Q3：数据里有错误怎么办？

没关系，数据清洗是我们系统的工作。你只需提供原始数据，我们会处理缺失值、异常值、单位统一等问题。

### Q4：涉及隐私和商业机密吗？

我们只关注设备运行数据（功率、电压、电流、发电量等），不涉及客户经营数据、财务数据、个人隐私。如仍有顾虑，可在合同中约定数据使用范围和保密条款。

### Q5：能不能直接对接厂家云 API？

可以。如果厂家提供开放 API 接口（如华为 FusionSolar API、阳光 iSolarCloud API），我们可以直接对接，省去手动导出步骤。需要提供 API 账号和接口文档。
